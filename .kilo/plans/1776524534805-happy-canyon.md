# Plan to Fix Employee Profile Loading Issue - UPDATED

## Problem Description
**ALL employees** fail to load their individual profiles. Both "View Employee" and "View Documents" actions result in "Employee not found" and redirect back to the directory. Employee directory listing works correctly, but individual profile lookup fails for every employee.

## Key Findings
- ✅ Employee directory lists employees successfully (list API works)
- ❌ Individual employee profiles fail to load (detail API fails)
- ❌ Happens for ALL employees, not just specific ones
- ✅ User is logged in as admin (permissions shouldn't be the issue)
- ❌ Employee IDs appear valid but profile lookup fails

## Root Cause Analysis

### CONFIRMED: API Server Error (500 Internal Server Error)
Browser console shows:
```
GET /api/employees/[id]?... 500 (Internal Server Error)
```

This confirms a **server-side code exception**, not a data or permission issue.

### Most Likely Causes (in order of probability):

1. **Database Query Error**: Issue in `EmployeeService.getEmployee()` - malformed SQL, missing table references, or enum casting errors

2. **TypeScript/Type Errors**: Runtime type errors when processing query results or building response

3. **Missing Dependencies**: Database schema changes not properly reflected, or missing enum values

4. **Async/Promise Handling**: Unhandled promise rejections in the query chain

## Investigation Steps

### Step 1: Add Server-Side Error Logging
- Add comprehensive try-catch blocks and logging to the API route
- Log all parameters, query execution, and error details
- Identify exactly where the 500 error is thrown

### Step 2: Debug EmployeeService.getEmployee()
- Add detailed logging throughout the method
- Log query construction, parameter binding, and results
- Test individual query components (user lookup, joins, etc.)

### Step 3: Check Database Schema Compatibility
- Verify all enum values match between schema and code
- Check table references and column names
- Ensure foreign key constraints are properly defined

### Step 4: Test API Isolation
- Call the API directly with different employee IDs
- Test with minimal parameters (no includes)
- Isolate whether the error is in base query or related data loading

## Solution Options

### Option A: Fix Database Query Error (Most Likely)
If there's a malformed query or missing references:
- Correct SQL syntax in `EmployeeService.getEmployee()`
- Fix table/column references
- Handle enum casting properly

### Option B: Fix Type Errors
If TypeScript/runtime type issues:
- Add proper type guards and validation
- Handle null/undefined values correctly
- Fix enum type mismatches

### Option C: Handle Missing Dependencies
If schema/code mismatches:
- Update enum definitions to match database
- Ensure all required tables exist
- Fix foreign key relationships

### Option D: Improve Error Handling
Regardless of root cause:
- Add comprehensive error logging with stack traces
- Return meaningful error messages instead of 500
- Add fallback handling for corrupted data
