# Plan: Fix Dashboard and Workflow Issues

## Overview
The user has reported several critical issues across the dashboard, workflow builder, and leave management system. This plan addresses all identified problems systematically.

## Issues Identified

### 1. Dashboard Branch Selection Problem
**Issue**: Dashboard filters show branch selector but doesn't display company's branches
**Location**: `components/dashboard/dashboard-filters.tsx`, `app/dashboard/schedules/page.tsx`
**Root Cause**: Potential tenant context issue or API not returning branches properly

### 2. Operations Dashboard Missing Schedules
**Issue**: `dashboard/operations` scheduled tab shows placeholder instead of actual operational schedules
**Location**: `components/dashboard/operations/scheduled-tab.tsx`
**Root Cause**: Component is just a placeholder, needs implementation

### 3. Workflow Settings Event Triggers
**Issue**: "Configure scheduling and role assignments" modal lacks clear UI for creating event-based triggers
**Location**: `components/builder/workflow-settings-modal.tsx`
**Root Cause**: Event triggers section exists but may need better UX for creating new events

### 4. Workflow Builder Missing Features
**Issue**: Missing text options for instructions, category selection, and geolocation options
**Location**: `components/builder/toolbox.tsx`, `components/builder/property-editor.tsx`
**Root Cause**: 
- 'instruction' step type exists but may not allow rich text
- Category selection missing from step configuration
- 'OPSLocationField' exists but may not be properly exposed

### 5. Template Manager Issues
**Issue**: Template cards show categories but filtering doesn't work; delete option missing
**Location**: `app/dashboard/builder/templates/template-manager.tsx`
**Root Cause**: Category filter exists but may not work; delete functionality implemented but possibly broken

### 6. Leave Management API Error
**Issue**: `TypeError: db.sql is not a function` in leave requests API
**Location**: `app/api/leave/requests/route.ts:58`
**Root Cause**: Incorrect usage of `db.sql` instead of imported `sql` function
